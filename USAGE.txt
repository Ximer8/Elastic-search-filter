🚀 БЫСТРЫЙ СТАРТ:
═══════════════════════════════════════════════════════════════════════════════

Шаг 1: Установите зависимости
─────────────────────────────────────────────────────────────────────────────
pip install requests --break-system-packages


Шаг 2: Запустите базовое сканирование
─────────────────────────────────────────────────────────────────────────────
./scan.sh normal targets.csv

ИЛИ напрямую:

python3 es_advanced_scanner.py -i targets.csv


Шаг 3: Проанализируйте результаты
─────────────────────────────────────────────────────────────────────────────
./scan.sh analyze es_results.json

📊 ЧТО ДЕТЕКТИРУЕТ СКАНЕР:
═══════════════════════════════════════════════════════════════════════════════

🔴 CRITICAL (Severity 9-10):
  ├─ credentials      - API keys, tokens, secrets
  ├─ passwords        - Пароли и хэши
  ├─ pii             - Email, телефоны, адреса
  ├─ medical         - Медицинские данные (HIPAA)
  └─ financial       - Платежи, карты, транзакции

🟠 HIGH (Severity 7-8):
  ├─ support_chats    - Диалоги с клиентами
  ├─ internal_notes   - Внутренние заметки
  ├─ production      - Production окружение
  └─ cloud_metadata  - AWS/GCP/Azure данные

🟡 MEDIUM (Severity 6-7):
  ├─ corporate       - Корпоративные данные
  ├─ backups         - Бэкапы и дампы
  ├─ cicd            - Jenkins, GitLab, CI/CD
  └─ auth_logs       - Логи аутентификации


📂 ФОРМАТЫ ВХОДНЫХ ДАННЫХ:
═══════════════════════════════════════════════════════════════════════════════

Сканер поддерживает различные форматы CSV:

1. Простой список IP:
   192.168.1.100
   10.0.0.50

2. IP:PORT:
   192.168.1.100:9200
   10.0.0.50:9300

3. Полные URL:
   http://192.168.1.100:9200
   https://elastic.example.com:9200

4. CSV с колонками (Shodan/Censys):
   ip,port,link
   192.168.1.100,9200,http://192.168.1.100:9200

5. Смешанный:
   192.168.1.100
   http://example.com:9200
   10.0.0.50:9300


📈 ПОНИМАНИЕ SEVERITY SCORE:
═══════════════════════════════════════════════════════════════════════════════

Severity Score = сумма баллов всех найденных детекций

  Score  80-100  🔴  CRITICAL      - Немедленное действие
  Score  50-79   🔴  HIGH          - Срочное внимание
  Score  30-49   🟠  MEDIUM        - Требует проверки
  Score  10-29   🟡  LOW           - Наблюдение
  Score  0-9     🟢  MINIMAL       - Минимальный риск


🔧 TROUBLESHOOTING:
═══════════════════════════════════════════════════════════════════════════════

┌─ Проблема: "No targets found" ────────────────────────────────────────────┐
│  Решение: Проверьте формат CSV, попробуйте --delimiter ";"               │
└────────────────────────────────────────────────────────────────────────────┘

┌─ Проблема: Connection timeout ────────────────────────────────────────────┐
│  Решение: Увеличьте -t (timeout), уменьшите -w (workers)                 │
└────────────────────────────────────────────────────────────────────────────┘

┌─ Проблема: Медленное сканирование ────────────────────────────────────────┐
│  Решение: Увеличьте -w до 100, уменьшите --sample-size                   │
└────────────────────────────────────────────────────────────────────────────┘


💡 ПОЛЕЗНЫЕ КОМАНДЫ ДЛЯ РАБОТЫ С РЕЗУЛЬТАТАМИ:
═══════════════════════════════════════════════════════════════════════════════

# Топ 10 самых критичных
cat es_results.txt | grep "🔴" | head -10

# Все хосты с credentials
grep "credentials" es_results.txt

# Подсчет по severity
cat es_results.txt | grep -o "score=[0-9]*" | cut -d= -f2 | sort -n | uniq -c

# Фильтр только production
grep "production" es_results.txt | sort -t= -k3 -nr

# Экспорт в CSV для Excel
cat es_results.txt | grep -v "^#" | sed 's/\t/,/g' > results.csv

# С использованием jq для JSON:
cat es_results.json | jq '.[] | select(.severity_score > 50)'
